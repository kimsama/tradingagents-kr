# TradingAgents — File Structure

A reference map of what lives where, and *why*. Pair this with [`00.Architecture.md`](00.Architecture.md), which explains the runtime relationships between these modules.

> **Convention.** This doc treats the repo root as the package root. Paths are written without a leading `./` unless that disambiguates a duplicate name (e.g. `cli/main.py` vs `main.py`).

### Top-level files at a glance

| File | Purpose | Notes |
|---|---|---|
| `pyproject.toml` | Source of truth for deps, package layout, pytest markers, console script | `tradingagents = "cli.main:app"` is what makes `tradingagents` work after `pip install .` |
| `uv.lock` | Pinned dependency graph from `uv sync` | Regenerate by running `uv lock`; do not hand-edit |
| `requirements.txt` | Single line `.` | Lets `pip install -r requirements.txt` install the local package; modern installs should prefer `uv sync` or `pip install .` |
| `Dockerfile` | Builder stage installs into `/opt/venv`, runtime stage copies it into `python:3.12-slim` | `ENTRYPOINT` is `docker-entrypoint.sh`, default `CMD` is `tradingagents` |
| `docker-entrypoint.sh` | Ensures `${HOME}/.tradingagents/{cache,logs,memory}` exist + writable, drops privileges via `setpriv` if running as root | POSIX `sh` (not bash) — keep portable |
| `main.py` | Programmatic example: build a `TradingAgentsGraph` from `DEFAULT_CONFIG` and run it once | Useful as a copy-paste starting point for embedding the framework |
| `test.py` | Ad-hoc yfinance smoke at the repo root | **Not** picked up by pytest (`testpaths = ["tests"]`); leave it to operators |
| `_diag_anthropic.py`, `_smoke_step5.py` | Local debugging harnesses | Both match the `_diag_*.py` / `_smoke_*.py` patterns in `.gitignore` and should never be committed |

### Why some boundaries look the way they do

- **`agents/` = factories, not classes.** Every agent module exposes a `create_<role>(llm)` function that returns a node callable. This keeps agents stateless from the graph's perspective and makes them trivial to test (pass a `MagicMock` LLM, call the returned function with a state dict).
- **`agents/utils/*_tools.py` are re-exports**, not data implementations. The actual vendor calls live in `dataflows/`. This split keeps tool annotations near the agents and HTTP/parsing near the data — see [`00.Architecture.md` §7](00.Architecture.md#7-data-layer--tradingagentsdataflows).
- **`agents/utils/agent_utils.py`** holds the language-instruction helper, the Korean localisation registry, and the role-label dictionary. It's a "shared util" file, not a domain module.
- **`graph/setup.py` vs `graph/trading_graph.py`.** `GraphSetup` only knows how to wire nodes; `TradingAgentsGraph` knows about config, memory log, checkpointing, and run lifecycle. Splitting them lets tests instantiate the graph wiring without paying the cost of building LLM clients.
- **`llm_clients/oauth/` is its own subpackage** because OAuth has its own credential format, refresh policy (delegated to source CLIs), and HTTP plumbing. Keeping it next to the rest of the provider code makes the dependency direction obvious — the OpenAI/Anthropic clients import from `oauth/`, not the other way round.
- **`dataflows/alpha_vantage_*.py`** is split per category so that adding a new endpoint or fixing a quota bug touches a small file, not a 1000-line catch-all.

## 3. The CLI front-end — `cli/`

```
cli/
├── __init__.py
├── main.py                 # Typer app — `tradingagents` console script entrypoint (~1250 LOC)
├── utils.py                # Interactive questionary prompts (ticker, date, language, models, …)
├── models.py               # CLI enums (AnalystType, etc.)
├── stats_handler.py        # Live LLM/tool/token counters fed into Rich panels
├── config.py               # CLI-side config defaults (separate from DEFAULT_CONFIG)
├── announcements.py        # Optional release-notes banner shown at startup
└── static/
    └── welcome.txt         # ASCII-art splash bundled via [tool.setuptools.package-data]
```

`cli/main.py` is intentionally large — it owns the eight-step interactive wizard, the live Rich layout (Progress / Messages / Current Report / Footer), report-saving prompts, and stat collection. If you change report layout, save paths, or report rendering, this is the file to touch.

```
└── test_google_api_key.py         # Google client uses the right env var
```

Pytest config (in `pyproject.toml`):

| Setting | Value | Why |
|---|---|---|
| `testpaths` | `["tests"]` | The root-level `test.py` and `_smoke_*.py` are *not* collected |
| `addopts` | `-ra --strict-markers` | Show short summary; refuse unknown markers |
| `markers` | `unit`, `integration`, `smoke` | Unit tests must avoid network/API calls; the others may require keys |

Provider clients are imported lazily so the suite runs cleanly without any provider's API key set — see `llm_clients/factory.py` and `tests/conftest.py`.

## 8. Generated / ignored directories

These do not live in version control but appear in working trees:

| Directory | Created by | Safe to delete? |
|---|---|---|
| `.venv/` | `uv venv` / `python -m venv` | Yes — recreate with `uv sync` |
| `.pytest_cache/` | pytest | Yes |
| `tradingagents.egg-info/` | `pip install -e .` / setuptools | Yes |
| `**/__pycache__/` | CPython | Yes |
| `~/.tradingagents/` | first run via Docker entrypoint or `default_config.py` | **No** — holds the memory log, checkpoints, cached reports |

The persistent `~/.tradingagents/` tree:

```
~/.tradingagents/
├── cache/              # data_cache_dir — vendor response cache + LangGraph checkpoints
│   └── checkpoints/
│       └── <TICKER>.db # per-ticker SqliteSaver database
├── logs/               # results_dir — saved reports + state JSONs
│   └── <TICKER>/
│       ├── <DATE>/
│       │   ├── reports/*.md
│       │   └── message_tool.log
│       ├── TradingAgentsStrategy_logs/
│       │   └── full_states_log_<DATE>.json
│       └── manual_reports/
│           └── <TICKER>_<TIMESTAMP>/complete_report.md
└── memory/
    └── trading_memory.md   # Append-only markdown decision log
```

Override locations with the env vars `TRADINGAGENTS_CACHE_DIR`, `TRADINGAGENTS_RESULTS_DIR`, `TRADINGAGENTS_MEMORY_LOG_PATH` (read by `default_config.py`).

## 9. Where to put new code

| You're adding … | Put it in | Touch points |
|---|---|---|
| A new agent role | `tradingagents/agents/<group>/<role>.py` | Re-export from `agents/__init__.py`; node + edge in `graph/setup.py`; routing predicate in `graph/conditional_logic.py` if branching |
| A new LLM provider | `tradingagents/llm_clients/<provider>_client.py` | Branch in `factory.py`; entry in `model_catalog.py`; validator in `validators.py` |
| A new data vendor | `tradingagents/dataflows/<vendor>*.py` (split per category if it has > 200 LOC) | Re-export from `dataflows/__init__.py`; dispatcher in `dataflows/interface.py`; option in `default_config.py` `data_vendors` |
| A new tool exposed to agents | `tradingagents/agents/utils/<category>_tools.py` | Re-export from `agents/utils/agent_utils.py`; register on the appropriate `ToolNode` in `graph/trading_graph.py:_create_tool_nodes` |
| A new structured-output schema | `tradingagents/agents/schemas.py` | Add render helper in same file; bind via `bind_structured` in the agent factory |
| A new test | `tests/test_<topic>.py` | Mark with `@pytest.mark.unit` (default) or `integration` / `smoke` |
| A one-off operator script | `scripts/<verb>_<target>.py` if shared, otherwise `_diag_*.py` / `_smoke_*.py` at the root if local |
| A new doc | `docs/NN.<Topic>.md` (numbered prefix) for ordered reading; `docs/<TOPIC>.md` for reference docs |
