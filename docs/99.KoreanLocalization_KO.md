## 2. 한국어 출력 이중 계층 설계

이 PR의 가장 중요한 설계 결정은 **한국어 출력을 두 계층으로 분리**한 것입니다.

### 2.1 계층 ①: 프롬프트 레벨 지시

각 사용자 대면 에이전트의 프롬프트 끝에 `get_language_instruction()`이 반환하는 문자열을 덧붙입니다.

```python
# agents/utils/agent_utils.py
def get_language_instruction() -> str:
    lang = get_config().get("output_language", "English")
    if lang.strip().lower() == "english":
        return ""           # 기본값일 때 추가 토큰 없음
    return f"\n\nWrite your entire response in {lang}."
```

이 지시는 다음 에이전트의 프롬프트에 추가되었습니다.

- Bull / Bear Researcher
- Research Manager
- Trader (system message)
- Aggressive / Conservative / Neutral Debator
- (기존부터 적용된) Market / Social / News / Fundamentals Analyst

> 내부 추론 텍스트는 영어로 진행될 수 있지만, 사용자에게 보이는 보고서와 최종 결정은 선택한 언어로 생성됩니다.

### 2.2 계층 ②: 후처리 라벨 치환

LLM이 생성한 산문은 그대로 두고, **결정형 라벨(스키마 라벨, 역할명, 헤딩 등)만 한국어로 치환**합니다. 후처리는 `localize_report_markdown(text, lang)`이 담당하며, 표시·저장 시점에만 적용됩니다.

치환 대상은 크게 네 분류입니다.

| 분류 | 예시 (영어 → 한국어) |
| --- | --- |
| 평가형 라벨 + 등급 | `**Recommendation**: Buy` → `**추천 의견**: 매수` |
| 텍스트 라벨 | `**Rationale**:` → `**근거**:` / `**Stop Loss**:` → `**손절가**:` |
| 화자 prefix | `Bull Analyst:` → `강세 분석가:` |
| 보고서 헤딩 | `## II. Research Team Decision` → `## II. 리서치 팀 결정` |

특별 케이스인 `FINAL TRANSACTION PROPOSAL: **BUY**` 도 `최종 거래 제안: **매수**` 로 변환됩니다.

### 2.3 왜 두 계층인가?

| 이유 | 설명 |
| --- | --- |
| **에이전트 상태는 영어 유지** | `AgentState`의 스키마 라벨이 영어여야 파서, 메모리 로그, 라우팅이 안정적으로 동작 |
| **LLM 비결정성 회피** | 라벨은 후처리로 100% 일관성 보장, 산문은 LLM이 생성 |
| **새 언어 추가 용이** | `_MARKDOWN_LOCALIZATIONS["<lang>"] = {...}` 한 줄로 일본어/중국어 등 확장 가능 (후속 리뷰 반영) |

## 3. 토론 라우팅 수정 (라우팅 안정화)

### 변경 전

```python
# 표시 텍스트의 prefix로 다음 노드를 결정 — 한국어 모드에서 동작 불가
if state["investment_debate_state"]["current_response"].startswith("Bull"):
    return "Bear Researcher"
```

이 코드는 `current_response`가 `"Bull Analyst: ..."`로 시작한다고 가정합니다. 한국어 모드에서는 이 prefix가 `"강세 분석가: ..."`로 변환되므로 `startswith("Bull")` 검사가 실패하고, Bull → Bull로 잘못 라우팅되어 `KeyError: 'Bull Researcher'` 가 발생했습니다.

### 변경 후

```python
# tradingagents/graph/conditional_logic.py
debate_count = state["investment_debate_state"]["count"]
if debate_count >= 2 * self.max_debate_rounds:
    return "Research Manager"
if debate_count % 2 == 1:
    return "Bear Researcher"
return "Bull Researcher"
```

스키마 레벨의 `count` 필드 패리티로 다음 발화자를 결정합니다. 표시 텍스트와 무관하므로 라벨 언어가 바뀌어도 라우팅이 영향받지 않습니다.

> 위험 토론(Aggressive/Conservative/Neutral)의 3-way 회전은 여전히 `latest_speaker.startswith("Aggressive" / "Conservative")` 를 사용합니다. 이 prefix들은 한국어로 치환되지 않는 내부 스키마 문자열이므로 안전합니다.

## 4. 리포트 저장과 표시

### 4.1 자동 저장 위치

`cli/main.py`의 `default_report_save_path()`가 추가되어, 분석 완료 후 저장 프롬프트의 기본 경로가 다음으로 표준화되었습니다.

```
${results_dir}/manual_reports/<TICKER>_<YYYYMMDD_HHMMSS>/
```

이전에는 `Path.cwd() / "reports" / ...` 였으며, 컨테이너에서는 쓰기 권한 문제가 발생할 수 있었습니다.

### 4.2 라이브 리포트 패널

CLI 좌측 "Current Report" 패널과 최종 "Complete Analysis Report" 출력에서 사용하던 영어 헤딩이 모두 `get_report_label("...")` 호출로 대체되어, 출력 언어가 한국어일 때 자동 한국어로 표시됩니다.

| 영어 키 | 한국어 표시 |
| --- | --- |
| `analyst_team_reports` | 애널리스트 팀 보고서 |
| `research_team_decision` | 리서치 팀 결정 |
| `trading_team_plan` | 트레이딩 팀 계획 |
| `risk_management_team_decision` | 리스크 관리 팀 결정 |
| `portfolio_manager_decision` | 포트폴리오 매니저 결정 |

### 4.3 저장된 마크다운 파일

`save_report_to_disk()`가 모든 섹션 헤딩과 역할명에 `get_report_label()` + `localize_report_markdown()`을 적용해, `complete_report.md`도 한국어 출력 모드에서 한국어 라벨을 사용합니다.

## 5. Docker 운영 변경

### 5.1 새 진입점 스크립트

`docker-entrypoint.sh`(신규)가 컨테이너 시작 시 다음을 수행합니다.

1. `${HOME}/.tradingagents/{cache,logs,memory}` 디렉터리 생성 (`appuser` 소유)
2. `TRADINGAGENTS_{CACHE,RESULTS}_DIR` / `TRADINGAGENTS_MEMORY_LOG_PATH` 환경변수 존중
3. root로 실행되는 경우 `setpriv`로 `appuser`로 권한 강하
4. `-`로 시작하는 인수 또는 알 수 없는 명령어는 자동으로 `tradingagents` 앞에 붙임

### 5.2 Dockerfile 변경

- `HOME=/home/appuser` 명시
- 빌드 시 `~/.tradingagents/{cache,logs,memory}` 디렉터리 생성 + `chown`
- `ENTRYPOINT`를 `/usr/local/bin/docker-entrypoint.sh`로 변경
- `CMD`는 `tradingagents` (기본 명령)

### 5.3 Persistent volume

`docker-compose.yml`의 양쪽 서비스(`tradingagents`, `tradingagents-ollama`)가 `tradingagents_data` named volume을 `/home/appuser/.tradingagents`에 마운트하여, 컨테이너 재생성 후에도 보고서·캐시·메모리 로그가 보존됩니다.

## 6. 부수 안정화 변경 (한국어와 직접 관련은 없으나 같은 PR에 포함)

### 6.1 Anthropic 클라이언트

| 변경 | 이유 |
| --- | --- |
| `max_tokens` 기본값을 4096으로 캡 | LangChain 기본 max_tokens가 매우 커서 rate limit 유발 |
| SDK `max_retries=0` | 자체 가시적 재시도와 SDK 숨은 재시도가 충돌하지 않도록 |
| 30초/90초 자체 재시도 (sync `invoke` + async `ainvoke`) | `429 rate_limit_error` 발생 시 자동 재시도 (Retry-After 헤더 존중) |
| `effort` kwarg 제거 | Anthropic이 OpenAI 스타일 effort 파라미터를 거부; `anthropic_effort` 설정은 예약(reserved)으로 표시 + 사용 시 경고 로깅 (후속 리뷰에서 추가) |

### 6.2 OAuth scope 검증

`tradingagents/llm_clients/oauth/openai_oauth.py`의 `_validate_model_request_scope()`가 추가되어, Codex JWT의 `scp` 배열 또는 `scope` 문자열에 `model.request` scope가 없으면 다음과 같은 친화적 오류를 발생시킵니다.

> "Codex ChatGPT token does not include the OpenAI API 'model.request' scope required for model calls. Set OPENAI_AUTH_MODE=api_key and provide an OPENAI_API_KEY..."

### 6.3 공통 rate-limit 헬퍼

`tradingagents/rate_limit.py`(신규)에 두 함수가 정의됩니다.

```python
is_rate_limit_error(exc) -> bool       # 상태 코드 / 클래스명 / 텍스트 fallback
retry_after_seconds(exc) -> float|None # Retry-After 헤더 파싱
```

이 헬퍼는 Anthropic 클라이언트 재시도 루프 + 구조화 출력 폴백 정책 양쪽에서 공유됩니다.

### 6.4 구조화 출력 폴백 차단

`agents/utils/structured.py`의 `invoke_structured_or_freetext()`가 429 에러 발생 시 자유텍스트 폴백을 **수행하지 않도록** 변경되었습니다. 같은 프롬프트를 재전송하면 quota 압박이 심해지기 때문입니다.

## 10. 한국어 출력 사용 방법 요약

```bash
# .env에서 출력 언어와 인증 모드 설정
export OUTPUT_LANGUAGE=Korean   # CLI에서도 Step 3에서 선택 가능
export OPENAI_AUTH_MODE=api_key
export OPENAI_API_KEY='sk-...'

# CLI 실행 (Docker 또는 로컬)
docker compose run --rm tradingagents
# 또는
tradingagents analyze
```

CLI Step 3 "Output Language"에서 `Korean (한국어)`를 선택하면, 위 모든 변경사항이 자동 적용되어 보고서·라이브 패널·저장 파일이 한국어 라벨로 출력됩니다.

자세한 운영 가이드는 [`10.USER_MANUAL_KO.md`](10.USER_MANUAL_KO.md)를 참조하세요.
